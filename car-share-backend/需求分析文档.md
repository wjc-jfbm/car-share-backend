# 面向粉丝团购场景的拼车协作与可信结算小程序 - 需求分析文档

## 一、项目概述

### 1.1 项目背景

随着粉丝文化的发展，粉丝团购已成为一种重要的消费形式。团购模式通常采用"多版本+随机周边"的销售方式，粉丝为了集齐所有版本和周边，往往需要购买多次，成本较高。

**拼车（Carry）** 是粉丝社区中流行的一种团购协作方式：
- 多位粉丝共同购买一整套商品
- 按成员或版本分配，每人获得特定部分
- 降低成本，提高集齐率

但目前拼车主要依赖微信群、微博等社交平台，存在以下问题：
- 信息分散，难以找到合适的拼车
- 缺乏信任机制，容易出现跑单
- 账目管理混乱，结算不便
- 缺乏标准化流程，沟通成本高

### 1.2 项目目标

构建一个面向粉丝团购场景的拼车协作与可信结算平台，实现：
- 🚗 便捷的拼车发起与参与
- 💰 透明的费用分摊与凭证管理
- 🔄 清晰的物流追踪与分配
- ✅ 可靠的信任保障与评价体系

### 1.3 项目定位

| 维度 | 描述 |
|------|------|
| **目标用户** | 粉丝群体（15-35岁） |
| **核心价值** | 让团购拼车更安全、更透明、更高效 |
| **差异化亮点** | AI智能匹配、智能分配、信用评价体系、物流一体化 |
| **技术特色** | uni-app跨平台、Spring Boot企业级架构、MyBatis-Plus高效ORM |

### 1.4 可行性分析

**市场需求**：
- 粉丝经济市场规模庞大，团购需求旺盛
- 拼车需求刚性，目前缺乏专业协作平台

**技术可行性**：
- 微信小程序生态成熟，开发成本低
- 规则评分算法可落地
- 凭证登记模式安全可靠

**工程价值**：
- 解决了真实的协作信任问题
- 涉及分布式资源分配算法
- 具有良好的工程实践价值

## 二、功能需求

### 2.1 用户角色与权限

| 角色 | 权限描述 |
|------|---------|
| **普通用户** | 发起拼车、参与拼车、评价车主、查看物流 |
| **车主** | 管理拼车订单、确认成员、发起结算、分配物品 |
| **管理员** | 用户管理、订单审核、系统配置、数据统计 |

### 2.2 核心功能模块

#### 2.2.1 用户管理模块
- 用户注册与登录（微信授权）
- 个人资料完善（偏好设置、收货地址）
- **信用积分体系**：基于交易记录的信用评分
- 黑名单管理

#### 2.2.2 拼车广场模块
- **AI偏好匹配**：根据成员/版本/小卡偏好推荐拼车
- **成团成功率预测**：基于规则评分预测拼车成功概率
- 拼车列表展示（筛选、排序）
- 拼车详情查看
- 一键参与拼车

#### 2.2.3 拼车发起模块
- 选择商品信息（专辑/周边等）
- 设置拼车规则（价格、分配方式）
- 设置截止时间
- 发布拼车

#### 2.2.4 拼车管理模块
- 成员认领：成员选择想要的版本/小卡
- 费用分摊：自动计算每人应分摊费用
- 支付凭证上传：成员上传付款截图/凭证
- 车主确认收款凭证

#### 2.2.5 物流追踪模块
- 物流信息录入
- 实时物流追踪
- **智能分配**：根据成员偏好自动分配版本/小卡
- 收货确认

#### 2.2.6 结算确认模块
- 生成结算账单
- 凭证审核
- 确认收款完成
- 退款处理

#### 2.2.7 评价体系模块
- 交易评价（车主/成员双向）
- 信用积分更新
- 评价展示

#### 2.2.8 消息通知模块
- 拼车状态变更通知
- 物流更新提醒
- 结算提醒

### 2.3 业务流程图

```
拼车发起流程：
选择商品 → 设置规则 → 发布拼车 → 成员报名 → 截止报名 → 确认订单 → 上传凭证 → 车主确认

拼车参与流程：
浏览广场 → 选择拼车 → 选择偏好 → 提交申请 → 车主确认 → 上传付款凭证

结算流程：
到货确认 → 智能分配 → 确认分配 → 生成账单 → 上传凭证 → 确认收款
```

## 四、AI能力集成

### 4.1 智能偏好匹配

**匹配对象**：成员/版本/小卡偏好

**算法：规则评分系统**

| 匹配因素 | 权重 | 说明 |
|----------|------|------|
| 成员偏好匹配 | 0.4 | 用户想要的成员与拼车可提供成员的吻合度 |
| 版本偏好匹配 | 0.3 | 用户想要的专辑版本与拼车可提供版本的吻合度 |
| 小卡偏好匹配 | 0.3 | 用户想要的小卡与拼车可提供小卡的吻合度 |

**匹配度计算**：
```
匹配度 = (成员偏好匹配 × 0.4) + (版本偏好匹配 × 0.3) + (小卡偏好匹配 × 0.3)
```

### 4.2 成团成功率预测

**预测因素**：

| 因素 | 权重 | 说明 |
|------|------|------|
| 历史成团率 | 0.3 | 该车主历史拼车的成功比例 |
| 当前参与率 | 0.3 | 当前已参与人数/目标人数 |
| 价格竞争力 | 0.2 | 价格相比市场均价的优势 |
| 信用评分 | 0.2 | 车主的信用积分等级 |

**预测公式**：
```
成团成功率 = (历史成团率 × 0.3) + (当前参与率 × 0.3) + (价格竞争力 × 0.2) + (信用评分归一化 × 0.2)
```

### 4.3 智能分配算法

**问题描述**：如何公平分配商品版本和小卡？

**解决方案**：
- **随机分配**：基于随机数生成，公平公正
- **偏好优先**：优先满足成员勾选的偏好
- **历史记录**：参考用户历史获得情况，避免重复

## 五、安全与风控

### 5.1 资金安全保障

| 措施 | 说明 |
|------|------|
| **凭证登记模式** | 不直接处理真实资金，仅登记付款凭证 |
| **凭证审核** | 车主审核成员上传的付款凭证 |
| **信用体系** | 交易评价影响信用积分 |
| **黑名单机制** | 不良用户拉黑 |

- 用户数据加密存储
